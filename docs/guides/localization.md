Phase 1: Planning & Audit

**Before writing any code or translations**

Complete these steps in order:

**1. Read the localization guide**

Review the comprehensive Greek localization guide included in Part 3 of this document.

**2. Audit the codebase**

Identify where all text strings live (hardcoded vs. components vs. config files).

**3. Create a translation plan**

Document each string\'s location and proposed Greek translation.

**4. Present translations section-by-section**

Get approval for each section before implementation.

Translation Guidelines (Quick Reference)

1\. **Formality:** Use formal \"εσείς\" throughout

2\. **Keep in English:** Optimems, +SolarControl, +Mind, Dashboard, Blog, AI, Demo

3\. **Use Greek energy terms:** ΑΠΕ (RES), ΔΕΔΔΗΕ (DSO), ΑΔΜΗΕ (TSO)

4\. **Avoid literal translations:** Don\'t translate idioms like \"game-changing\", \"out of the box\", \"powerhouse\" word-for-word

5\. **Maintain punchy tone:** Don\'t make sentences longer than the English

6\. **Names stay unchanged:** Erik Lindqvist, Maria Papadopoulos, etc.

7\. **Country names translate:** Sweden → Σουηδία, Germany → Γερμανία, France → Γαλλία, Greece → Ελλάδα

Phase 2: Implementation (After Review)

Once all translations are approved:

1\. Determine i18n approach (locale files OR inline replacement)

2\. Implement translations following the approved translation document

3\. Test that all Greek text renders correctly (character encoding, line breaks, etc.)

4\. Flag any edge cases (text overflow, layout issues with longer Greek strings)

Part 4: Implementation Notes

Translation Principles

**1. Formality Level**

Use formal \"εσείς\" (you-plural/formal) throughout all customer-facing content. This maintains professionalism while being approachable.

**2. Brand Names & Product Names**

ALWAYS keep these in English: Optimems, +SolarControl, +Mind

**3. Common Tech Terms**

These can stay in English as they\'re commonly used: Dashboard, Software, Hardware, Demo, Blog, AI

**4. Avoid Literal Translations**

English marketing phrases need cultural adaptation:

• \"Game-changing\" → \"Καινοτόμος\" or \"Επαναστατικός\" (not \"Αλλάζει το παιχνίδι\")

• \"Out of the box\" → \"Έτοιμο προς χρήση\" (not \"Έξω από το κουτί\")

• \"Powerhouse\" → \"Δυναμική πηγή\" (not \"Σπίτι δύναμης\")

• \"Seamless\" → \"Απρόσκοπτη\" (not \"Χωρίς ραφές\")

Character Encoding & Typography

Ensure UTF-8 encoding throughout the codebase to properly display Greek characters. Pay special attention to:

• Accented vowels: ά, έ, ή, ί, ό, ύ, ώ

• Diacritics: Greek uses tonos (΄) for stress

• Punctuation: Greek uses semicolon (;) as question mark

Layout Considerations

Greek text may be 10-30% longer than English. Test these areas for overflow:

• Navigation buttons

• CTA buttons

• Card titles

• Mobile viewport widths

Before launching the Greek version:
